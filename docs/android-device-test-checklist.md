# Android 真机验收清单

## 准备

- 使用 Android 8/10/12 与 Android 13/14/15/16 两个权限世代中至少各一台真机。
- 准备包含收件、已发送、服务号、验证码、营销短信、彩信图片的测试数据；不要使用真实银行卡或登录验证码。
- 大数据量设备建议准备至少 5,000 条短信和 10,000 张图片；不足时至少验证分页请求与虚拟节点数量。
- 启动 `server`，或明确只验证本地排队行为。

## 首次启动和短信权限

- [ ] 首次启动不会自动读取短信。
- [ ] 点击“授权并一键同步全部”后才出现短信、照片/视频和联系人权限弹窗。
- [ ] 拒绝权限后页面保持可用，且没有读取短信。
- [ ] 再次允许权限后自动扫描历史短信并安排上传，不需要再点击“立即补扫”。
- [ ] 扫描包含收到和已发送短信，草稿/失败等类型可以查看但不会错误上传为新收到短信。

## 授权后无人值守收集

- [ ] 首次授权并退出 App 后，不点击“立即补扫”，另一台手机发送短信，服务器 TXT 自动出现 `inbox` 记录。
- [ ] 从最近任务划掉 App 后再次接收短信，仍能自动写入本地队列并上传。
- [ ] 手机重启后不打开 App，收到新短信时仍能自动排队和上传。
- [ ] 启动 App 时自动补扫收件和已发送短信，不需要手动操作。
- [ ] 等待 WorkManager 周期任务后，新产生的已发送短信以 `sent` 方向进入 TXT。
- [ ] 已发送短信在下一次周期补扫前被立即删除时允许缺失，这是非默认短信应用的系统边界。
- [ ] 收件广播、启动补扫和周期补扫重叠时，同一短信仍只产生一条服务器记录。

## 全量入队与旧过滤迁移

- [ ] 淘宝、拼多多、验证码、退订、促销、优惠券等短信都进入“等待上传”或“已备份”。
- [ ] 首页不存在黑名单入口和“已过滤”计数。
- [ ] 从含旧 `filtered_records` 的版本升级后重新扫描，原先被过滤的短信能够进入队列。
- [ ] 重复扫描不会重复增加相同 `recordId` 或内容指纹。

## 密码与十分钟会话

- [ ] 首页“查看全部短信”入口可以正常打开。
- [ ] 错误密码、空密码或带空格密码提示“密码错误”，且原生层没有读取短信。
- [ ] 固定密码 `88888888` 解锁后显示会话列表。
- [ ] 切后台、锁屏、进入详情再返回不会立刻锁定，也不会续期。
- [ ] 解锁满 10 分钟或手动点击“锁定”后，短信、联系人和图片缓存清空，必须重新输入密码。
- [ ] 分页请求返回期间会话到期时，旧响应不会重新显示敏感数据。

## 联系人备注、号码和头像

- [ ] 首次解锁会单独请求联系人权限；拒绝时不影响短信备份和查看。
- [ ] 拒绝联系人权限时显示完整号码和统一默认 PNG，不显示首字头像。
- [ ] 允许联系人权限后，备注名、号码、号码标签和系统头像与系统联系人一致。
- [ ] 同一联系人在会话列表和会话详情使用相同头像、备注和号码。
- [ ] `+86 138...`、`138...` 等不同保存格式仍能匹配同一联系人。
- [ ] 没有系统头像的联系人使用默认头像，透明边缘没有绿色色边或拉伸。
- [ ] Node 服务端记录中不包含联系人备注、号码标签或头像 URI。

## 全部、收到、已发送

- [ ] 三个分类首次只显示首批记录，继续下滑每次再读取 40 条。
- [ ] 列表按时间从新到旧，连续翻页没有重复或明显漏项。
- [ ] “收到”只显示收件，“已发送”只显示已发送，“全部”包含草稿、发件箱、失败、排队等系统记录。
- [ ] 正文完整换行显示，方向、短信/彩信、时间、已读、已查看和 SIM 标签正确。
- [ ] 加载超过 200 条后页面仍能滚动，JS 短信窗口不无限增长。
- [ ] 快速切换“全部/收到/已发送”时，旧请求不会覆盖当前分类。

## 会话详情

- [ ] 点击会话进入气泡详情，收到在左、发送/发件箱/排队在右。
- [ ] 顶部复用会话列表联系人头像、备注、手机号与号码标签。
- [ ] 每条记录显示正文、方向、记录时间、发送时间、类型/状态、已读/已查看、SIM、短信中心和系统记录 ID。
- [ ] 到达顶部每次加载更早 40 条，加载后原来的首条消息位置不突然跳走。
- [ ] SMS 与 MMS 同时间记录连续翻页不重复；彩信附件点击可以预览。
- [ ] 彩信缩略图使用延迟加载，未点击前不会一次打开所有原图。

## 一万张图片性能

- [ ] 未授权图片权限时短信和会话仍可用。
- [ ] Android 13+ 请求 `READ_MEDIA_IMAGES`；Android 8–12 请求 `READ_EXTERNAL_STORAGE`。
- [ ] 授权后先显示相册与数量，不出现长时间白屏或一次性加载所有图片。
- [ ] 进入 10,000 张图片相册后首屏可滚动，快速拖动数十屏不闪退、不明显冻结。
- [ ] 切换相册后旧相册图片不会混入新相册。
- [ ] 使用布局检查确认同时存在的图片节点约为视口加缓冲行，而不是 10,000 个。
- [ ] 使用 Android Studio Profiler/系统内存观察确认图片滚动后内存趋于稳定；页面最多缓存四页元数据。
- [ ] 点击图片预览只带当前附近图片，预览不会因构造 10,000 个 URI 卡住。
- [ ] 锁定或会话到期后返回图片页需要重新加载，旧缩略图不残留。

## 服务器控制的图片与视频同步

- [ ] Android 13+ 同时请求 `READ_MEDIA_IMAGES` 和 `READ_MEDIA_VIDEO`；Android 8–12 使用 `READ_EXTERNAL_STORAGE`。
- [ ] Android 14+ 只选择部分照片/视频时，只同步被系统授权的项目，设备结果明确显示部分完成。
- [ ] 管理网页把回溯小时改为 `24` 后，新指令的绝对时间窗为服务器当前时间往前 24 小时。
- [ ] 再改为 `2` 小时后，新指令不再上传 2 小时以前的图片或视频，证明手机端没有写死 7 天。
- [ ] 所有已授权相册都参与扫描，而不是只读取相机相册；图片和视频类型都能出现在网页控制台。
- [ ] 一万张图片设备执行时，每批清单不超过 100 条，内存没有随总相册数量持续增长。
- [ ] 大视频使用 1 MiB 分块，断网后服务器保留 `.part`，恢复网络后从已有偏移继续。
- [ ] 同时上传的媒体不超过 2 个；视频上传期间页面仍可退出，系统显示 SmsBackup 数据同步通知。
- [ ] 相同媒体再次同步不会生成第二份文件；服务端媒体文件缺失时会要求手机重新上传。
- [ ] 服务器设置“仅 Wi-Fi”后，移动网络不上传媒体，切换 Wi-Fi 后自动继续。

## 服务端手动获取与自动获取

- [ ] 浏览器打开 `/admin`，错误密码无法进入，正确 `SMS_ADMIN_PASSWORD` 登录后可以查看设备。
- [ ] App 首次连接后设备名称、设备 ID、App 版本和最后在线时间出现在控制台。
- [ ] App 不在前台但未被强行停止时，点击“立即获取”后手机在下一次 WorkManager 轮询中领取指令。
- [ ] 每条服务器指令都会先补扫全部收到和已发短信，再处理服务器时间窗内的图片和视频。
- [ ] 自动同步开启时按服务器周期产生新指令；关闭后只响应管理员手动创建的指令。
- [ ] 修改小时数不会改变已经排队指令的时间窗，下一条新指令使用新值。
- [ ] Node 重启后动态设置、待处理指令、设备状态、短信去重和媒体索引从 TXT 恢复。

## 新短信、进程回收和强行停止

- [ ] App 前台收到短信后等待上传数量增加。
- [ ] 从最近任务划掉 App 后收到短信，Manifest Receiver 仍能建立队列记录。
- [ ] `adb shell am kill <实际包名>` 模拟普通进程回收后，新短信仍能建立记录。
- [ ] 系统设置“强行停止”后应用不会自行恢复；再次手动打开后恢复任务，这是 Android 预期限制。
- [ ] 强行停止期间产生但仍保留在系统短信库中的收件和已发送短信，重新打开后能够通过立即补扫进入队列。

## 网络和 Node 服务端

- [ ] 未配置服务器时短信保留在等待上传队列。
- [ ] 错误地址“测试连接”明确失败且短信不丢失。
- [ ] HTTP 地址只有开启“允许 HTTP”后才能保存和测试。
- [ ] 默认地址为 `http://119.91.65.202:8787`，App 令牌与服务器 `SMS_BACKUP_TOKEN` 一致。
- [ ] 错误或缺失 Bearer Token 访问 `POST /api/sms`、`GET /api/sms`、`GET /api/sms/export.md` 时返回 `401`。
- [ ] 恢复服务器和网络后 WorkManager 自动重试。
- [ ] 后端返回 `2xx` 后记录从等待上传变为已备份。
- [ ] 后端返回 `5xx` 或断网时保持待上传并使用退避重试。
- [ ] 同一 `recordId` 重复发送时，`server/data/sms-records.txt` 只写一次。
- [ ] 记事本记录包含方向、号码、正文、时间、设备和 SIM 等上传字段。
- [ ] `GET /api/sms` 可以按 `limit`、`offset`、`deviceId` 和 `direction` 分页筛选。
- [ ] `GET /api/sms/export.md` 下载的 UTF-8 Markdown 包含完整正文，正文中的换行和标签不会破坏文档结构。
- [ ] 公网 HTTP 抓包能够看到明文风险提示；正式长期使用前配置 HTTPS。

## 重启与隐私

- [ ] 手机重启后 WorkManager 任务仍保留，网络恢复后继续上传。
- [ ] 15 分钟周期补漏即使被系统延迟，也不会产生重复记录。
- [ ] 应用图标、权限和同步状态始终可见。
- [ ] Logcat、HBuilderX 控制台和普通日志中没有短信正文。
- [ ] 持有人可以在系统设置撤销短信、联系人、图片权限或卸载应用。

## 本轮不执行

- [ ] 不运行 HBuilderX 云打包。
- [ ] 不生成或替换 APK；完成以上真机验收后再单独打包。
