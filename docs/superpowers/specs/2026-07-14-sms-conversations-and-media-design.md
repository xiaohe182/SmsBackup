# 短信会话与媒体浏览设计

## 目标

将现有的受密码保护短信列表升级为一个 Android 短信浏览器：验证密码后，可在固定十分钟的内存会话中浏览会话、完整短信、MMS 图片附件和经用户授权的本机相册；用户点击“锁定”或会话到期后必须重新输入密码。

## 范围

- 会话从首次成功解锁起固定十分钟，不因刷新、切页、返回首页或切后台而续期。
- 手动锁定和到期会清除内存中的密码、短信、MMS 附件和相册索引；不会写入本地持久化存储。
- 主浏览页提供“会话”“全部”“收到”“已发送”“图片”分页。
- 会话按 Android `thread_id` 分组；缺失 `thread_id` 时按地址分组。列表显示号码、最近内容、时间和未读状态；点击进入会话详情。
- 会话详情使用 Android 风格的左右气泡、日期分隔线、系统无衬线字体、完整正文和 MMS 图片缩略图；点击图片可调用全屏预览。
- “图片”页同时显示 MMS 图片附件和本机相册照片，并按相册分组筛选；相册照片仅在用户授予图片读取权限后显示。
- 不申请通讯录权限：会话标题使用系统短信库返回的号码或地址，避免扩大权限范围。

## 安全与会话

`SmsViewerSession` 是唯一持有查看密码的 TypeScript 内存模块。首次验证密码后记录 `unlockedAt` 与 `expiresAt = unlockedAt + 10 * 60 * 1000`。所有短信、MMS 和相册原生读取都必须通过该模块取得仍有效的密码；无效时，模块先清除敏感缓存，再让页面显示密码页。

页面隐藏、返回首页和应用转入后台都不主动锁定。页面重新显示、调用刷新、切换详情页或尝试预览前均根据真实时间检查有效期，因此后台停留超过十分钟时会在回到应用后立即锁定。每个已解锁页显示剩余时间和“锁定”按钮。

原生 Kotlin 入口继续逐次校验固定密码 `88888888`，而不信任前端会话状态。密码只存于进程内存，应用被系统终止后自然失效。

## 数据与原生接口

现有 SMS 查询继续返回全部已发送和收到的短信及元数据，并新增统一的媒体响应：

- `getViewerMessagesJson(password)` 返回完整 SMS 记录，按时间倒序。
- `getMmsMessagesJson(password)` 查询 `Telephony.Mms`，读取消息箱、线程、地址、时间、正文和 `content://mms/part/<id>` 图片附件 URI。MMS 的日期秒值会转换为毫秒。
- `getGalleryPhotosJson(password)` 查询 `MediaStore.Images.Media`，返回照片 URI、相册 ID/名称、显示名、拍摄时间和 MIME 类型。
- `requestMediaPermissions()` 在 Android 13+ 请求 `READ_MEDIA_IMAGES`，在 Android 8–12 请求 `READ_EXTERNAL_STORAGE`；读取接口也会分别检查权限。

原生响应保留 `authorized` 与对应权限状态。无效密码时不查询系统 Provider；缺失权限时返回空数组和 `permissionGranted=false`。MMS 图片与相册照片使用 `content://` URI，不复制到应用私有目录，也不上传。

## 页面结构

1. `pages/messages/messages` 是受保护入口。它负责密码验证、会话倒计时、SMS/MMS/相册权限提示、数据刷新，以及五个分页的汇总视图。
2. `pages/conversation/conversation` 仅通过非敏感的会话键定位内存会话中的数据。它显示同一会话内按时间顺序的 SMS 与 MMS，包含所有正文和可预览图片。
3. `domain/sms-viewer-session` 负责会话时钟、敏感缓存、会话聚合、分页过滤和用于路由的稳定会话键。
4. `services/sms-backup` 扩展其可测试的类型与适配器，统一解析消息、MMS 和照片响应；不可用平台返回空数据和 `authorized=false`。

## 错误处理

- 密码错误：保留密码界面，清空临时输入，提示错误，不查询任何 Provider。
- 读取短信权限未授予：显示短信授权操作，不显示短信正文。
- 图片权限未授予：短信正常可浏览；“图片”页显示单独的图片授权操作，不显示相册照片。MMS 文本照常显示。
- 原生响应格式错误或读取失败：保留有效会话，清空该类数据，显示可重试错误；不会将旧数据误标为最新数据。
- 会话在异步读取期间到期：丢弃读取结果并立刻回到密码界面。

## 验证

- 单元测试覆盖固定十分钟不续期、手动锁定、到期清理、按会话分组、分页筛选和媒体权限选择。
- 服务测试覆盖 SMS/MMS/相册响应解析、无效响应和不可用平台回退。
- 页面与原生契约测试覆盖新增页面路由、锁定控制、分页、权限声明、MMS Provider 和 MediaStore 查询。
- 完整检查运行 `npm test -- --run`、`npm run type-check` 与 `npm run build:app-plus`。
