# Android 高兼容后台定位与轨迹分析设计

## 目标

在现有 Android-only SmsBackup 中加入由用户显式启停的轨迹记录、联系人查看和设备健康诊断。开启轨迹后使用常驻定位前台服务，在应用退到后台、锁屏或从最近任务划掉后继续采样；每三分钟最多写入一个经纬度点，并在本机完成漂移过滤、异常跳点剔除、距离和实际路线分析。

## Android 边界

- 使用系统 `LocationManager` 的 GPS 与网络定位，不依赖 Google Play 服务，覆盖华为及其他无 GMS 设备。
- 定位服务必须从可见页面、在权限授予后启动；Android 10+ 声明 `foregroundServiceType="location"`，Android 14+ 声明 `FOREGROUND_SERVICE_LOCATION`。
- 前台服务显示不可静默隐藏的持续通知，并提供停止操作；服务使用 `START_STICKY` 与 `android:stopWithTask="false"`。
- 普通应用无法绕过系统“强行停止”，也无法保证在厂商深度冻结、关机或权限被撤销后继续运行。页面提供电池优化、应用详情和厂商自启动说明。
- 不申请后台定位权限：持续采样依靠用户可见的定位前台服务；这样减少权限范围并适配 Android 新版审核规则。

## 数据流

1. 用户进入轨迹页，授予精确/近似定位和通知权限。
2. 页面调用原生入口启动 `LocationTrackingService`。
3. 服务监听 GPS 与网络位置；首点立即保存，之后只有距离上次保存时间至少三分钟的最佳候选点才落盘。
4. `LocationDatabase` 保存轨迹会话和点位，记录纬度、经度、精度、海拔、速度、方向、来源与采集时间。
5. TypeScript 服务读取原生 JSON；`route-analysis` 按会话分组，过滤低精度点、静止漂移和不可能跳点，计算总距离、持续时间、平均速度、移动时间和可绘制折线。
6. 轨迹页显示运行状态、最近坐标、路线统计、折线预览，并可调用系统地图查看最新位置。

## 联系人与设备信息

- 联系人使用 `ContactsContract.CommonDataKinds.Phone`，仅在用户授予 `READ_CONTACTS` 后读取姓名、号码、标准化号码、联系人 ID 和可用头像 URI。
- 联系人不写入应用数据库、不上传，只在受密码保护的十分钟页面会话内保留。
- 设备诊断读取电池电量、充电状态、健康、温度、电压、技术类型、制造商、型号、Android/API 版本、总/可用内存、总/可用存储、网络连接和电池优化状态。
- 不读取通话记录、系统账号、其他应用私有数据或其他没有明确产品用途的敏感数据。

## 兼容与可靠性

- 采样间隔固定为 `180000ms`；服务向系统请求更频繁的候选位置，以便从 GPS/网络中选择更准确的点，但 SQLite 最多每三分钟写一次。
- 数据库使用事务、时间和会话索引；服务重启后恢复当前会话，不重复创建会话。
- 位置提供者关闭、权限撤销和服务启动失败会转化为状态信息，不使应用崩溃。
- WorkManager 继续负责短信上传与每日补漏，不用于三分钟定位调度。
- 页面每次显示时刷新原生状态；应用进程重建后仍可读取已落盘轨迹。

## 隐私

- 轨迹仅存于应用私有 SQLite，不写普通日志、不上传服务器、不嵌入短信数据。
- 清除轨迹需要先停止定位；清除操作删除全部轨迹会话和点位。
- 持续通知明确显示正在采集位置，用户可以在应用或通知中停止。

## 验证

- 纯函数测试覆盖 Haversine 距离、三分钟序列、精度过滤、跳点过滤和会话分组。
- TypeScript 服务测试覆盖状态与点位 JSON 解析、不可用平台回退。
- 原生契约测试覆盖权限、前台服务声明、持续通知、`START_STICKY`、双定位提供者、三分钟节流、SQLite 表和无敏感日志。
- 页面契约测试覆盖轨迹入口、权限、启停、系统设置、路线分析和地图打开。
- 联系人和设备契约测试覆盖显式权限、受保护读取、电池字段、内存/存储/网络字段和无持久化缓存。
- 运行全量 Vitest、`vue-tsc --noEmit` 与 `uni build -p app-plus`。
