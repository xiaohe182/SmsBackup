# 受保护短信与图片浏览

首页选择“查看全部短信”，输入固定密码 `88888888` 后进入受保护浏览器。前端校验密码后，Android 原生入口会在读取系统 Provider 前再次校验。

查看密码与服务器 Bearer Token 是两个独立用途：默认值目前都为 `88888888`，查看密码保护本机页面，Bearer Token 保护 Node 的短信写入、分页读取和 Markdown 导出。服务器令牌可以通过 App 设置与 `SMS_BACKUP_TOKEN` 同步修改。

## 自动备份与浏览的关系

- 设备持有人首次授予短信权限后，新收到短信通过 Manifest Receiver 自动入队，不要求打开浏览页面。
- 收件和已发送短信会在授权、应用启动、手机重启恢复和最短 15 分钟周期任务中补扫。
- 浏览器的十分钟锁定只清理查看会话内存，不关闭后台备份。
- 非默认短信应用没有已发送短信广播；发送后在下一次补扫前立即删除的记录无法保证捕获。
- 系统“强行停止”会暂停所有自动路径，重新打开 App 后立即补扫并恢复任务。

## 十分钟内存会话

- 会话从解锁时开始固定有效 10 分钟，刷新、返回、进入详情和切到后台都不会续期。
- 点击“锁定”或会话到期后，密码、联系人身份、会话摘要、短信分页和图片缓存会一起从内存清除。
- 页面不把短信正文、联系人备注或头像 URI 写入前端持久化存储。
- 固定密码只能防止普通误看，不能替代设备锁、真实用户认证或加密存储。

## 联系人身份

解锁时应用单独请求 `READ_CONTACTS`。允许后，会话列表和会话详情复用同一份系统联系人信息：

- 联系人备注；
- 完整手机号或短信地址；
- 手机、住宅、自定义标签等号码类型；
- 系统联系人缩略头像 URI。

联系人号码在 Android IO 线程中通过 `PhoneNumberUtils` 标准化，并兼容 `+86`、`86` 与本地号码形式。联系人数据只保存在当前十分钟会话内，不写数据库、不加入短信上传请求，也不发送到 Node 服务端。

拒绝联系人权限不会影响短信备份或查看。页面会保留完整号码，并使用统一的 `src/static/default-contact.png`，不会再用名字或号码首字充当头像。

## 短信与彩信分页

- “会话”只接收轻量汇总，不包含整段历史正文。
- “全部”“收到”“已发送”每次默认读取 40 条，原生接口强制限制在 1–100 条。
- 会话详情按 `threadId` 分页合并 SMS/MMS；到达顶部再读取更早 40 条。
- 前端短信窗口最多保留 200 条，避免长时间翻页后持续增加 JS 内存。
- 稳定游标由 `timestamp`、`kind`、`sourceId` 组成，相同时间的短信和彩信不会因页边界重复。
- 详情展示完整正文、方向、记录时间、发送时间、类型、发送状态、已读/已查看、SIM、短信中心、系统记录 ID 和彩信附件。

所有短信 Provider 查询都通过 `UTSAndroid.getDispatcher('io')` 执行。页面不再调用旧的全量短信、全量彩信接口。

## 相册分页与一万张图片

- 图片页先读取相册名称、数量与封面 URI，不打开位图。
- 当前相册每次默认读取 60 条元数据，原生接口强制限制在 1–120 条。
- 虚拟网格只创建视口加上下两行缓冲范围内的图片节点；一万条元数据不会生成一万个 `<image>`。
- 图片页最多缓存四个数据页，切换相册、锁定或会话到期立即释放。
- 滚动事件按 16ms 合并，快速滑动时旧相册请求通过 `mediaGeneration` 丢弃，不能覆盖当前相册。
- 图片使用 `lazy-load="true"`；全屏预览最多传入当前附近四页 URI，不构造一万条预览数组。

Android 8–12 使用 `READ_EXTERNAL_STORAGE`，Android 13 及以上分别使用 `READ_MEDIA_IMAGES` 和 `READ_MEDIA_VIDEO`，Android 14+ 兼容用户只选择部分照片/视频。拒绝媒体权限时短信和联系人会话仍可正常使用；服务器相册同步只处理系统实际授权、且位于服务器时间窗内的图片和视频。本地查看页仍只分页展示图片，不会因为后台同步而一次加载整个相册。

## 厂商兼容路径

Android 8 及以上优先使用 `QUERY_ARG_LIMIT`、`QUERY_ARG_OFFSET` 和结构化排序参数。原生层检查 `EXTRA_HONORED_ARGS`；厂商 MediaStore 未声明支持 offset 时，重新使用受控 Cursor 跳过与截取。无论走哪条路径，单次只构建一页 JSON，不构建全量图片数组。

## 本轮验证边界

自动测试覆盖 10,000 张图片的可见范围、四页 LRU 淘汰、短信页去重、十分钟敏感数据清理、联系人字段和页面契约。`npm run build:app-plus` 已使用编译器 4.87 完成资源构建。本轮没有执行 DCloud 云打包或生成 APK；真实厂商 Provider 和位图内存表现仍按真机清单验收。
