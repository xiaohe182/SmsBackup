# 动态时间范围相册媒体与服务端拉取设计

## 目标

SmsBackup 在现有全部收发短信自动同步基础上，增加相册图片与视频备份、服务端手动请求和服务端自动请求能力。

服务端继续保持零数据库、Node.js 18+ 可直接运行。全部服务端代码、管理网页、配置、说明和运行依赖必须位于 `server` 目录；部署时只复制完整的 `server` 文件夹即可。

“服务端拉取”定义为：Node 记录同步请求，Android App 在后台轮询到请求后，读取已经获得用户授权的系统数据并上传。服务器不能绕过 Android 权限，也不能从公网直接连接通常处于 NAT 后面的手机。

## 已确认范围

- 短信：每次请求补扫手机短信库中的全部收到和已发送短信，沿用现有 `recordId` 去重。
- 相册：同步系统 `MediaStore` 中所有获得授权的相册，包括图片和视频，不局限于“相机”相册。
- 时间范围：由服务器动态配置回溯小时数，默认 `168` 小时。Android App 不写死七天。
- 手动触发：Node 管理页面为指定设备提供“立即获取”按钮。
- 自动触发：Node 按设备保存自动同步开关和周期；手机至少每十五分钟轮询一次。
- 本机触发：App 首页提供“一键同步全部”，但执行时仍先向服务器领取当前配置。
- 存储：短信、媒体元数据、设备状态、配置和请求使用 UTF-8 TXT/JSON Lines；图片和视频保持原始二进制文件。
- 部署边界：服务端不得引用 `server` 目录之外的模块、静态文件或配置。

## 方案选择

采用“服务端指令队列 + Android WorkManager 轮询”。它不依赖 Google Play 服务，适合国产 Android 设备。周期任务受 Android 省电和厂商策略影响，服务端点击后通常在零到十五分钟内被手机领取，但不承诺精确到达时间。

不采用 FCM 和常驻 WebSocket：前者在国内网络与无 Google 服务设备上兼容性不足；后者需要长期前台通知、耗电更高，仍可能被厂商系统限制。

## 服务器动态配置

Node 提供全局默认配置和单设备覆盖配置：

```json
{
  "mediaLookbackHours": 168,
  "autoSyncEnabled": true,
  "autoSyncIntervalMinutes": 15,
  "wifiOnly": false
}
```

- `mediaLookbackHours` 是正整数，允许范围 `1` 到 `87600` 小时，默认 `168`。
- 管理员可在网页中随时修改，无需重新打包或重装 App。
- 创建请求时，Node 使用服务器时间计算并固定 `windowStart`、`windowEnd` 和 `lookbackHours`。
- 已经领取的请求不会因后续配置变化而漂移；新请求使用最新配置。
- App 只按请求中的绝对时间窗查询图片和视频，不使用写死天数，也不依赖手机时钟计算范围。
- 配置写入 `data/settings.txt`，服务重启后从 TXT 重建。

## 数据流程

1. 管理员使用密码 `88888888` 登录 `GET /admin`。
2. 管理员设置回溯小时数、自动同步开关、自动周期和网络策略。
3. 管理员点击“立即获取”，或 Node 判断设备已达到自动同步周期。
4. Node 将带固定时间窗的幂等请求追加到 `data/sync-requests.txt`。
5. Android App 使用 Bearer Token 领取下一条请求。
6. App 先补扫全部收发短信，再查询请求时间窗内的图片和视频。
7. App 每批最多提交一百条媒体清单；Node 返回缺失项和未完成文件的续传偏移。
8. App 最多并发上传两个媒体文件，使用 `Content-Range` 分块流式传输。
9. Node 写入 `.part` 文件，完整接收后原子重命名并追加元数据。
10. App 提交结果；Node 在管理页面展示数量、字节、时间和错误。

手机离线、断网、重启或进程被回收时，请求保持待处理状态；下次任务只补传缺失内容。

## Android 客户端

### 权限适配

- Android 8–12 使用 `READ_EXTERNAL_STORAGE`。
- Android 13 及以上同时使用 `READ_MEDIA_IMAGES` 和 `READ_MEDIA_VIDEO`。
- Android 14 及以上适配 `READ_MEDIA_VISUAL_USER_SELECTED`。
- 用户选择“允许所有照片和视频”时才能覆盖配置时间窗内的全部媒体。
- 用户只授权部分媒体时，只上传系统授予的部分，并向服务器报告“部分媒体权限”。
- 每次任务开始时重新检查权限，不永久缓存权限结果。

### 相册与存储卷适配

- Android 10 及以上枚举系统提供的可用外部媒体卷，兼容主存储和可访问的 SD 卡。
- 旧版本回退到 `EXTERNAL_CONTENT_URI`。
- 覆盖相机、截图、下载、微信等应用创建的相册。
- 空相册名统一显示“未知相册”，使用媒体类型、媒体卷和 Bucket ID 保持身份稳定。
- 通过 `ContentResolver` 流式读取 `content://` URI，不依赖真实文件路径。
- 图片和视频分别分页查询，再按拍摄时间和媒体 ID 倒序归并。
- 优先使用 `DATE_TAKEN`；缺失时回退到 `DATE_ADDED * 1000`。
- 查询条件严格使用请求下发的 `windowStart` 和 `windowEnd`。

### 图片和视频格式

- 接受系统标记为 `image/*` 或 `video/*` 的媒体。
- 图片重点覆盖 JPEG/JPG、PNG、WebP、GIF、HEIC/HEIF、BMP、DNG。
- 视频重点覆盖 MP4、MOV、3GP、WebM、MKV、AVI。
- 不在手机端解码、旋转、压缩或转码，保持原始字节、EXIF、方向和视频内容。
- 动态照片保持原始媒体文件，不额外拆分其中可能包含的视频数据。
- 未知但合法的 `image/*` 或 `video/*` 仍允许上传，服务器使用安全扩展名并保留原 MIME。
- 中文、空格、Emoji 和特殊字符只作为 UTF-8 元数据，不参与服务器磁盘路径拼接。

### 后台任务与性能

- 复用现有 WorkManager，新增指令轮询和媒体上传 Worker。
- 周期轮询最短十五分钟，要求网络已连接。
- App 前台点击“一键同步全部”时创建一次性任务并立即领取服务器配置。
- 媒体清单分页读取，不一次构建全部数组。
- 上传并发固定为两个，单文件使用分块流式读取，不把整张图片或整段视频读入内存。
- 大文件使用 WorkManager 长任务和前台服务通知，明确显示备份正在进行。
- 中断后从服务器确认的字节偏移继续；成功文件不重复上传。
- 失败使用指数退避，单个媒体失败不阻断同批其他文件。
- 状态页分别显示待上传/已上传图片、待上传/已上传视频、传输字节、最近同步和错误。

## Node 服务端

### 自包含目录

```text
server/
  server.js
  package.json
  README.md
  MEDIA_SYNC_DESIGN.md
  public/
    admin.html
    admin.css
    admin.js
  lib/
    device-store.js
    settings-store.js
    sync-request-store.js
    media-store.js
  data/
    sms-records.txt
    media-records.txt
    devices.txt
    settings.txt
    sync-requests.txt
    media/
      <deviceId>/
        <yyyy-mm-dd>/
          <mediaId>.<extension>
```

现有模块继续位于 `server` 根目录；本功能新增模块固定放入 `server/lib`。所有模块均不得依赖 `server` 之外的文件。`data` 目录和 TXT 文件首次运行时自动创建。

### 接口

- `GET /api/health`：公开健康检查。
- `GET /admin`：管理页面。
- `POST /api/admin/session`：校验管理密码并签发短时内存会话。
- `GET /api/admin/devices`：设备状态列表。
- `GET /api/admin/settings`：读取全局和设备配置。
- `PUT /api/admin/settings`：动态修改回溯小时数和同步策略。
- `POST /api/admin/devices/:deviceId/sync`：按当前配置创建手动请求。
- `GET /api/device/:deviceId/commands/next`：App 领取下一条请求或自动生成到期请求。
- `POST /api/device/:deviceId/commands/:requestId/complete`：App 上报结果。
- `POST /api/media/manifest`：提交媒体清单并返回缺失项及续传偏移。
- `PUT /api/media/:mediaId/content`：使用 `Content-Range` 上传二进制分块。
- `GET /api/media`：按设备、时间和媒体类型分页读取元数据。
- `GET /api/media/:mediaId/content`：鉴权读取原文件，支持 HTTP Range。

健康检查之外的 API 使用 `Authorization: Bearer <SMS_BACKUP_TOKEN>`。管理密码默认也是 `88888888`，但服务端区分网页会话和 App Token。

### 存储和安全

- `media-records.txt` 每行一个 JSON 对象，记录设备、媒体类型、媒体 ID、相册、原显示名、拍摄时间、视频时长、MIME、大小、路径和接收时间。
- `sync-requests.txt` 追加记录创建、领取、进度、完成和失败事件，启动时重建内存索引。
- `settings.txt` 追加配置变更事件，最后一个有效值生效。
- Node 按请求中固定的时间窗校验媒体元数据，拒绝窗外图片和视频。
- 媒体唯一键包含设备 ID、媒体类型、媒体卷、MediaStore ID、修改时间和文件大小。
- 磁盘文件名只使用服务器生成的安全 ID 和受信 MIME 映射扩展名。
- 拒绝设备 ID、路径和文件名中的路径穿越、绝对路径和非法字符。
- 单文件上限由 `SMS_MEDIA_MAX_BYTES` 配置，默认 10 GiB。
- 分块先写同目录 `.part`；Node 重启后保留有效临时文件并返回续传偏移。
- 完整大小校验成功后原子重命名；不把 `.part` 当作成功文件。
- 管理页面输出短信、文件名、相册和错误信息时执行 HTML 转义。
- 当前 HTTP 会暴露 Token 和私人数据；公网长期使用应在 Node 前配置 HTTPS。

## 管理页面

管理页面是 `server/public` 下的纯 HTML、CSS 和 JavaScript，不使用构建工具或 CDN。页面包括：

- 密码登录。
- 全局回溯小时数和单设备覆盖值，可动态保存。
- 自动同步开关、周期和 Wi-Fi 策略。
- 设备名称、设备 ID、最后在线和最后成功同步。
- 短信累计数、时间窗内图片数、视频数、待上传数、字节和失败数。
- “立即获取”与“重新请求”按钮。
- 请求状态：等待手机、扫描中、上传中、完成、部分失败。
- 图片分页懒加载；视频只预加载元数据并按需播放，不在列表初次进入时请求全部原文件。

页面只调用受保护的 Node API，不直接读取磁盘 TXT。

## 错误处理

- 没有相册权限：短信继续同步，结果标记“短信完成、相册未授权”。
- Android 14 部分授权：上传可访问媒体并标记“部分媒体权限”。
- 手机离线：请求保持等待，不判定永久失败。
- 媒体在扫描后被删除、损坏或属于不可读取的云端占位文件：记录单项失败，其余继续。
- 文件重复：返回已存在，不重写。
- Node 重启：从 TXT 和 `.part` 文件重建状态并继续等待或续传。
- 磁盘空间不足：保留错误事件，不提交成功记录。
- Token 错误：返回 401，不泄露设备和文件信息。
- 配置值非法：返回 400，并保留上一次有效配置。

## 测试与验收

### Node 自动测试

- 管理认证成功、失败和会话过期。
- 动态小时数保存、非法范围拒绝、全局值和设备覆盖。
- 请求创建时正确固定服务器时间窗，配置修改不影响已领取请求。
- 自动请求到期生成、重复轮询幂等和重启恢复。
- 设备只能领取自己的请求。
- 媒体清单只返回缺失项或续传偏移。
- 图片和视频分块上传、HTTP Range、大小限制、续传和重复上传。
- 路径穿越、危险扩展名和特殊字符拦截。
- TXT 损坏行跳过并保留有效记录。
- `server` 复制到临时目录后可独立启动并完成接口测试。

### Android 自动测试

- App 使用服务器下发的任意合法小时数，不存在写死七天逻辑。
- 时间窗边界、`DATE_TAKEN` 回退和服务器绝对时间正确。
- 全部收到和已发送短信继续进入现有去重队列。
- 服务端请求触发短信补扫与相册同步。
- 图片和视频分页、最多两个并发、长任务通知、重试和续传。
- 主存储、可用 SD 卡、相机、截图、下载、应用相册和空相册名。
- 常见图片、视频及未知合法 `image/*`、`video/*` 原始流。
- 中文、空格、Emoji 文件名不会破坏 TXT、HTTP 或路径。
- 权限拒绝、部分授权、断网、重启和媒体被删除。

### 真机验收

- 管理页面将回溯值从 `168` 改为 `24` 后，新请求只上传服务器时间最近 24 小时的图片和视频，无需重装 App。
- 手机不打开 App 时，服务端请求能在后台轮询窗口内开始处理。
- TXT 出现全部去重短信，媒体目录只包含请求时间窗内文件。
- 重复请求不产生重复短信行或重复媒体文件。
- 大视频中断后能从已确认偏移续传。
- 上传数百张图片和视频时 App 可正常滚动、切后台且不明显卡顿。
- 强行停止 App 后后台任务不会运行；重新打开 App 后恢复，并在界面和文档中明确此限制。

## 不在本次范围

- 音频和其他非图片、非视频文件。
- 绕过 Android 权限读取短信或相册。
- FCM 秒级推送。
- WebSocket 常驻连接。
- 数据库、对象存储或第三方云盘。
- 远程删除手机数据。
